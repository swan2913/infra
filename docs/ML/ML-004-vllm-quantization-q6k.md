# ML-004 · vLLM 모델 양자화 Q4_K_M → Q6_K 업그레이드

> 작성일: 2026-04-25

## 배경

Carnice-9b 초기 배포 시 Q4_K_M을 사용했으나, RTX 3080(10GB VRAM) 여유분 분석을 통해
Q6_K 업그레이드가 가능함을 확인.

## VRAM 분석

| 항목 | Q4_K_M | Q6_K |
|------|--------|------|
| 모델 파일 크기 | 5.63 GB | 7.36 GB |
| 실측 총 VRAM (nvidia-smi) | 6,898 MiB | ~8,669 MiB (추정) |
| VRAM 여유 | ~3,342 MiB | ~1,571 MiB |
| Q8_0 가능 여부 | — | ❌ (KV 포함 시 초과) |

RTX 3080 VRAM: 10,240 MiB

## 아키텍처 특이사항 (GDN 하이브리드)

Carnice-9b는 순수 트랜스포머가 아닌 GDN(Gated DeltaNet) 하이브리드 구조:
- **8개 레이어**: 풀어텐션 → KV 캐시 사용 (Q4_0, 1,152 MiB)
- **32개 레이어**: GDN 순환 → 별도 상태 메모리 (f32, 100.5 MiB, 이미 최고 정밀도)

KV 캐시 양자화 업그레이드(Q4_0 → Q8_0)는 8개 레이어에만 적용되어 효과 제한적.
모델 가중치 자체의 양자화 업그레이드(Q6_K)가 전체 레이어에 적용되어 더 효과적.

## 품질 개선

| 양자화 | Perplexity 추가 손실 | 비고 |
|--------|-------------------|------|
| Q4_K_M | +0.0535 | 기존 |
| Q5_K_M | +0.0353 | 미제공 |
| **Q6_K** | **~+0.020** | 채택 — 6비트 |
| Q8_0 | ~+0.002 | VRAM 초과 |

Q6_K는 Q4_K_M 대비 perplexity 손실 약 34% 감소. 코딩/추론 태스크에서 유의미한 차이.

## 컨텍스트 설정 유지

변경 없음:
- `--ctx-size 131072` (슬롯당 65,536 tokens × 2슬롯)
- `--parallel 2`
- `--cache-type-k q4_0`, `--cache-type-v q4_0`
- `--flash-attn on`

Qwen3.5-9B 실효 컨텍스트 한계(~64K)와 현재 설정이 일치하므로 유지가 최적.

## 변경 파일

- `charts/vllm/values.yaml`: `model.file` Q4_K_M → Q6_K
